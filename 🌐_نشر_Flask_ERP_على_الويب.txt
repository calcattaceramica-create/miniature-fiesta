   - اضغط "New +" (أعلى اليمين)
   - اختر "Web Service"


الخطوة 3️⃣: ربط المستودع
────────────────────────────────────
🔗 اختر المستودع:
   calcattaceramica-create/miniature-fiesta

📝 إذا لم يظهر:
   - اضغط "Configure account"
   - امنح Render صلاحية الوصول


الخطوة 4️⃣: إعدادات التطبيق
────────────────────────────────────
Name:
   ded-erp-system

Region:
   Frankfurt (EU Central) - الأقرب للسعودية

Branch:
   main

Root Directory:
   (اتركه فارغاً)

Runtime:
   Python 3

Build Command:
   pip install -r requirements.txt && python init_database.py

Start Command:
   gunicorn --bind 0.0.0.0:$PORT --workers 2 --timeout 120 run:app


الخطوة 5️⃣: إضافة قاعدة البيانات
────────────────────────────────────
⚠️ مهم جداً!

في نفس الصفحة، انزل لأسفل:

📊 اضغط "Add Database"
   - اختر "PostgreSQL"
   - Name: ded-database
   - Database: ded_erp
   - User: ded_user
   - اضغط "Create Database"


الخطوة 6️⃣: المتغيرات البيئية
────────────────────────────────────
في قسم "Environment Variables":

اضغط "Add Environment Variable":

1. FLASK_ENV = production
2. SECRET_KEY = (اضغط Generate)
3. PYTHON_VERSION = 3.11.0

⚠️ DATABASE_URL سيتم إضافته تلقائياً!

🔧 حل المشاكل
═══════════════════════════════════════════════════════════════

❌ خطأ في البناء (Build Failed)?
────────────────────────────────────
✅ تحقق من:
   1. ملف requirements.txt موجود
   2. ملف run.py موجود
   3. لا توجد أخطاء في الكود


❌ خطأ في قاعدة البيانات?
────────────────────────────────────
✅ تحقق من:
   1. تم إنشاء PostgreSQL Database
   2. DATABASE_URL موجود في Environment Variables
   3. تم تشغيل init_database.py


❌ التطبيق لا يفتح?
────────────────────────────────────
✅ تحقق من:
   1. Status = "Live" في Render
   2. Logs لا تحتوي على أخطاء
   3. انتظر دقيقة إذا كان نائماً
